## 1-1. コンポーネント化＝レゴのパーツ作り

巨大な1枚のHTMLを、意味のある単位で切り分けます。

- **メリット:**
  - **読みやすい:** ファイルが短くなるから、どこに何があるかすぐわかる。
  - **使い回せる:** 一度作った「ボタン」を、他の場所でも何度でも使える。
  - **バグに強い:** 小さい単位でテストできるから、間違いに気づきやすい。

<!-- 
【講師用台本】
想像してみてください。
「プロフィールカード」が3箇所に出てくるサイトを作る時、毎回同じHTMLをコピーしますか？
もしデザインを少し変えたくなったら、3箇所全部直すのは大変ですよね。
コンポーネント化しておけば、元となる「部品」を一個直すだけで、全部の場所がパッと更新されます。
これこそが、プロがNext.jsを使う最大の理由なんです。
-->

## 2-1. コンポーネントの作り方

JavaScriptの関数を作って、JSXを `return` するだけ！

```tsx
// 1. 部品を作る（大文字で始めるのがルール！）
export function MyWidget() {
  return (
    <div className="bg-white p-4 rounded-xl shadow">
      <h3>お知らせ</h3>
      <p>今日は良い天気ですね。</p>
    </div>
  );
}

// 2. 使うときはタグのように呼び出す
<MyWidget />
```

<!-- 
【講師用台本】
コンポーネントを作るのは、実はとっても簡単です。
「普通のJavaScriptの関数」を作って、その中で表示したいHTML（JSX）を返すだけ。
唯一のルールは、名前を必ず「大文字」で始めること。
これによって、Reactは「あ、これはHTMLのタグじゃなくて、自作の部品なんだな」と判断してくれるんです。
-->

---

## 3-1. Props：部品への「指示書」

同じ部品でも、中身だけ変えたい時に使います。

- **Props (プロパティ):** 部品に外から渡すデータのこと。
```tsx
function SkillBadge({ name }) {
  return <span className="bg-blue-100 p-2">{name}</span>;
}

// 呼び出す時にデータを渡す
<SkillBadge name="React" />
<SkillBadge name="Next.js" />
```

<!-- 
【講師用台本】
「Props」は部品に対する「カスタマイズ指示」です。
例えば「スキルバッジ」という部品を作っても、中身の文字が全部同じだったら困りますよね。
使う時に `name="React"` と指定することで、見た目は同じだけど中身の違う、自分好みの部品をたくさん生み出すことができるんです。
-->

---

## 4-1. 繰り返しの新常識：`.map()`

Reactでリストを表示する時は、`forEach` ではなく `.map()` を使います。

- **理由:** `.map()` は、配列から「新しいJSXの配列」を作ってくれるから。

```tsx
const skills = ["HTML", "CSS", "JS"];

return (
  <ul>
    {skills.map((s) => (
      <li key={s}>{s}</li>
    ))}
  </ul>
);
```

<!-- 
【講師用台本】
JavaScriptの時に学んだ `forEach` ですが、React（JSX）の中では `.map()` という兄弟を使います。
「配列の中身を、一つずつliタグに変身させてね！」というイメージです。
これを使えば、データが増えても自動でリストが出来上がっていきます。
JSXの中でJavaScriptを動かす時は `{ }` で囲むのを忘れないでくださいね。
-->

---

## 5-1. 豆知識：`key` は React への「目印」

`.map()` を使う時、必ず `key` という属性をつけなければいけません。

- **なぜ？**
  - Reactは、どのデータが更新されたかを `key` で判断しているから。
  - これがないと、Reactが「どれを直せばいいのー！」とパニックになり、動きが遅くなってしまいます。
- **ルール:** 
  - 重複しない、ユニークな値（IDなど）を指定しましょう。

<!-- 
【講師用台本】
リストを作る時に絶対に忘れてはいけないのが `key` です。
Reactはとても賢いので、変更があった場所だけを効率よく書き換えます。
その時に「これは1番目のデータ、これは2番目のデータ…」と目印をつけるのが `key` の役割です。
コンソールに「Warning: Each child in a list should have a unique key...」と出たら、「keyを忘れてるよ！」というReactからの叫びだと思ってください。
-->

## 6-1. components フォルダで整理

部品が多くなってきたら、専用のフォルダを作りましょう。

- **`app/page.tsx`**: 全体の組み立て（司令塔）。
- **`components/`**: 個別の部品（兵隊たち）。

**💡 ヒント:** 
- `export function ...` で書き出し、`import { ... } from ...` で読み込む。

<!--

## 8. 迷った時のチェックリスト

1. **大文字:** コンポーネント名は **S**idebar のように大文字から始まっていますか？
2. **Import:** `import { Sidebar } from "../components/Sidebar"` は正しいですか？
3. **Return:** JSXは `return ( ... );` の中に書いていますか？
4. **Key:** `.map()` の中に `key={...}` はありますか？

<!-- 
【講師用台本】
