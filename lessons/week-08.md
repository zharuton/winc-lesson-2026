## 1-1. 大量のデータをどう並べる？

APIから返ってくるデータは、多くの場合「リスト（配列）」の形をしています。

- **例: ニュース記事リスト**
```json
[
  {"title": "新技術発表！", "url": "..."},
  {"title": "プログラミング入門", "url": "..."},
  {"title": "今日のランチ", "url": "..."}
]
```

- **悩み:** これを一つずつHTMLで書くのは大変…
- **解決策:** JavaScriptに「全部同じ形で並べて！」と命令する。

<!-- 
【講師用台本】
ニュースサイトやSNSを想像してください。
記事が100個あったら、100回HTMLを書くのでしょうか？ そんなわけありませんよね。
プロは「一つのひな形」を用意しておいて、JavaScriptに「このデータ100個分、ひな形に当てはめて表示して！」と命令します。
この「繰り返し」の考え方が、動的なWebサイトの核心なんです。
-->

## 2-1. ループの魔法：`forEach`

配列の中身を一つずつ順番に処理する命令です。

```javascript
const fruits = ["りんご", "バナナ", "ぶどう"];

fruits.forEach((item) => {
  console.log(`${item}を食べました！`);
});
```

**💡 イメージ:** 
- 配列という「ベルトコンベア」から、荷物（データ）を一つずつ取り出して処理する感じです。

<!-- 
【講師用台本】
繰り返し処理の代表格が `forEach` です。
「配列の中にあるもの全部に対して、この命令をやってね」という意味です。
一つ取り出しては処理し、また次を取り出しては処理する…
このシンプルな動きを組み合わせることで、どれだけ大量のデータも一瞬で画面に表示できるんですよ！
-->

---

## 3-1. テンプレートリテラル：HTMLのスタンプ

JavaScriptの中でHTMLの「ひな形」を作る便利な書き方です。

- **バッククォート `` ` `` で囲む:**
```javascript
const title = "Hello JS";
const html = `
  <div class="card">
    <h3>${title}</h3>
  </div>
`;
```

- **`${ }`**: 変数の中身をその場所に埋め込めます。

<!-- 
【講師用台本】
JavaScriptの中でHTMLを書きたい時、普通の文字列だとすごく読みにくいんです。
そこで使うのが「テンプレートリテラル」。
キーボードの「P」の横にあるバッククォートで囲みます。
`${ }` を使うことで、変数の内容をHTMLの中にスッと差し込むことができる、魔法のスタンプ台のような機能です。
-->

## 4-1. 【やってみよう 1】リストを自動生成しよう！

1. スキルの配列を作る。
2. `forEach` を使って、HTMLのリスト (`li`) を作って表示する。

```javascript
const skills = ["HTML", "CSS", "JS"];
const list = document.querySelector("#skill-list");

skills.forEach(s => {
  const html = `<li>学習中：${s}</li>`;
  list.insertAdjacentHTML("beforeend", html);
});
```

**✅ 成功イメージ:** 配列に書いたものが、画面に箇条書きで出たら成功！

## 5-1. 豆知識：APIキーって何？

有名なAPIを使う時、メールアドレスなどで登録するともらえる「秘密のパスワード」です。

- **なぜ必要なの？**
  - 「誰がどのくらいデータを使っているか」を管理するため。
  - 無制限に使われるのを防ぐため。
- **注意点:** 
  - 秘密の鍵なので、本来は他人に教えてはいけません！
  - 自分のサイトを公開する時は、鍵を隠す工夫が必要になります（Phase 3で学びます）。

<!-- 
【講師用台本】
APIの世界には「入場チケット」のようなものがあります。それがAPIキーです。
多くのサービスでは、このキーがないとデータを売ってくれません。
今日は練習用の公開APIを使いますが、本格的な開発では自分のキーを大切に扱うことがエンジニアの重要なマナーになります。
「鍵を出しっぱなしにしない」。これ、覚えておいてくださいね！
-->

## 6-1. おもてなしの心：Loading表示

インターネットの通信は、たまに数秒かかることがあります。

- **悪い例:** 真っ白な画面。ユーザーは「バグったかな？」と不安になります。
- **良い例:** 「読み込み中...」の文字や、グルグル回るアニメーション。

```javascript
display.textContent = "データを取ってきています...";
await fetch("...");
display.textContent = ""; // 終わったら消す
```

<!-- 
【講師用台本】
通信中の「待ち時間」をどう演出するか。これがアプリの使い心地を左右します。
「今、お使いに行っているから少し待ってね」というメッセージを出すだけで、ユーザーの安心感は全然違います。
fetchの前に「読み込み中」を表示し、終わったらそれを上書きする。
このひと手間が、あなたのサイトを「プロの作品」に変えるんですよ。
-->

## 8. 迷った時のチェックリスト

1. **配列の形:** データの正体は配列ですか？ `console.log` で確認！
2. **バッククォート:** `` ` ``（バッククォート）と `'`（シングルクォート）を間違えていませんか？
3. **`${ }`の中身:** 変数名が間違っていませんか？
4. **追加の場所:** `insertAdjacentHTML` の `"beforeend"` などの場所指定は正しいですか？

<!-- 
【講師用台本】
エラーで詰まったらここをチェック。
特によくあるのが、バッククォートの打ち間違いです。
「@」のキーにある記号ですが、これを使わないと `${ }` はただの文字として扱われてしまいます。
色が正しく変わっているか、エディタの画面をよく見てくださいね。
