## 1-1. JavaScript の役割

Web制作の「三種の神器」の一つ。

- **HTML:** 構造（家の骨組み、間取り）
- **CSS:** 装飾（壁紙の色、おしゃれな家具）
- **JavaScript:** 振る舞い（自動ドア、照明のスイッチ、お掃除ロボット）

**💡 JSができること:** 
- ユーザーの操作に合わせて中身を変える。
- 外部から情報を取ってくる。
- 複雑な計算をさせる。

<!-- 
【講師用台本】
HTMLが骨組み、CSSがお化粧なら、JavaScriptは「中身（振る舞い）」です。
「クリックしたら画面がパッと変わる」「時間が経つと背景が変わる」。
そういった「動き」はすべてJavaScriptの仕事です。
プログラミングらしく、少しロジカルな部分も出てきますが、自分の命令通りに画面が動いた時の感動は格別ですよ！
-->

## 1-2. どこに書くのが正解？

CSSと同じく、**「外部ファイル」**に書くのがプロの標準です！

1. `main.js` というファイルを新しく作る。
2. `index.html` の一番下に下の1行を書く。

```html
<script src="main.js" defer></script>
```

**💡 `defer` って？**
- 「HTMLを全部読み込み終わってから実行してね」という指示。
- これがないと、JSが先に動いてしまい「HTMLが見つからない！」とエラーになることがあります。

<!-- 
【講師用台本】
JavaScriptも専用のファイルを作ってあげましょう。
HTMLの最後に `<script>` タグで読み込みます。
ここで大事なのが `defer` というおまじない。
「HTMLがまだ準備できていないのに、JSが走り出しちゃった！」という事故を防ぐための、優しさの合言葉です。
-->

## 3-1. データの箱：変数 (`const`)

プログラムで使う情報を、名前のついた「箱」に保存します。

- **`const` (定数):** 一度入れたら変えない箱。
- **`let` (変数):** 後から中身を入れ替える箱。

```javascript
const name = "WINCくん";
console.log(name); // 「WINCくん」と表示される
```

**💡 豆知識:** 
- 今のプログラミングでは、**基本は `const`** を使い、どうしても変えたい時だけ `let` を使うのが安全だと言われています。

<!--

「自分の名前」や「今日の気温」など、何度も使いたいデータを箱に入れて名前をつけます。
名前をつけることで、後から「あの名前のデータを持ってきて！」とお願いできるわけですね。

## 4-1. DOM：HTMLを操るための「地図」

JavaScriptからHTMLを触るために、ブラウザはHTMLを「ツリー状の地図」に変換します。

- **DOM (Document Object Model)** と呼びます。
- **イメージ:** 
  - `document` が地図の全体。
  - 各タグ（`h1`や`p`）が地図上の地点。
- JavaScript はこの地図を見ながら、「h1地点を青色に変えろ！」と命令を出します。

<!-- 
【講師用台本】
ここでちょっとだけ理論のお話。「DOM（ドム）」です。
ブラウザは私たちが書いたHTMLを読み込むと、裏側で「扱いやすい地図」に作り変えてくれます。
JavaScriptはこの地図を頼りに、「2番目の段落を探して！」とか「画像の色を薄くして！」と指示を出すんです。
DOMは、JavaScriptとHTMLを繋ぐ架け橋なんですよ。
-->

## 5-1. 要素を探せ！ `querySelector`

地図の中から、特定のタグを捕まえる「最強の虫眼鏡」です。

- **使い方はCSSセレクタと同じ！**
```javascript
// IDが「title」の要素を捕まえる
const title = document.querySelector("#title");

// クラスが「description」の要素を捕まえる
const desc = document.querySelector(".description");
```

**💡 ポイント:** 
- 捕まえた要素を、まずは `const` で箱に入れておきましょう。

<!-- 
【講師用台本】
命令を出すには、まず「誰に」を決めないといけません。
そこで使うのが `querySelector` です。
中身の書き方はCSSと全く同じ。IDなら `#`、クラスなら `.` をつけます。
「君に決めた！」と、ターゲットを指定する感覚ですね。
-->

## 6-1. 【やってみよう 1】文字を書き換えよう！

捕まえた要素の中身を、JSの力で上書きします。

```javascript
// 1. 要素を捕まえる
const myName = document.querySelector("h1");

// 2. 中身を書き換える
myName.textContent = "JSで書き換えられた名前";
```

**✅ 成功イメージ:** HTMLに書いてある名前が、読み込んだ瞬間に別の名前に変わったら大成功！

<!--

リロードした瞬間に名前がパッと変わる様子は、まるで魔法ですよ！

## 6-2. 色や形もJSで自由自在

スタイルも直接いじることができます。

```javascript
const box = document.querySelector(".card");

// 背景色をオレンジに！
box.style.backgroundColor = "orange";

// クラスを追加してCSSを適用！
box.classList.add("highlight");
```

**💡 豆知識:** 
- `box.style` で直接いじるよりも、あらかじめCSSでクラスを作っておいて `classList.add` する方が、コードが綺麗になりますよ！

<!-- 
【講師用台本】
文字だけでなく、見た目も変えられます。
CSSでは `background-color` と書きましたが、JSでは `backgroundColor` と、ハイフンを消して大文字で繋ぐのがルールです。
後で「ボタンを押した時だけ色を変える」といった仕組みを作る時に、このテクニックがめちゃくちゃ役立ちます！
