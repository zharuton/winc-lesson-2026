## 1-1. 「イベント」はユーザーからの合図

ブラウザ上で発生するあらゆる「出来事」をイベントと呼びます。

- **クリック:** ボタンを押した！ (`click`)
- **入力:** フォームに文字を書いた！ (`input`)
- **マウス移動:** 要素の上にカーソルが乗った！ (`mouseenter`)
- **読み込み完了:** ページが表示された！ (`DOMContentLoaded`)

**💡 イメージ:** 
- ユーザーがブラウザという「呼び出し鈴」を鳴らすイメージです。

<!-- 
【講師用台本】
「イベント」と聞くとお祭りを想像するかもしれませんが、プログラミングの世界では「きっかけ」という意味です。
皆さんがスマホをタップしたり、スクロールしたりするたびに、ブラウザの中では「今、クリックイベントが発生したよ！」という通知が飛び交っています。
私たちはその通知を捕まえて、「じゃあ、このプログラムを動かしてね」と予約しておくわけです。
-->

## 2-1. 予約の窓口：`addEventListener`

特定の要素に、「この合図が来たら、これをして！」と予約します。

```javascript
const btn = document.querySelector("#btn");

// 「btn」が「click」されたら、「右側の命令」を実行して！
btn.addEventListener("click", () => {
  console.log("ボタンが押されました！");
});
```

**💡 書き方のルール:** 
- `()` の中は、`"イベント名"`, `動かしたい命令` の順番です。

<!-- 
【講師用台本】
予約の仕方はこうです。
まず、誰の合図を待つか（セレクタ）を決めます。
そして、`addEventListener` という長い名前の命令を使います。「イベントのリスナー（聞き手）を追加する」という意味ですね。
「クリックされたら、コンソールにログを出してね」といった予約を、一つずつ積み上げていきます。
-->

## 3-1. 命令をひとまとめに：関数 (Function)

何度も使う命令や、長い命令は「関数」として名前をつけて保存しておけます。

```javascript
// 「挨拶をする」という命令を「sayHello」という名前で保存
function sayHello() {
  alert("こんにちは！");
}

// 予約する時に名前を呼ぶだけ
btn.addEventListener("click", sayHello);
```

**💡 豆知識:** 
- 関数にすることで、コードがスッキリ読みやすくなります。

<!-- 
【講師用台本】
プログラムが長くなってくると、あちこちに同じことを書くのは大変です。
そこで「関数」の出番です。
「料理のレシピ」に名前をつけるようなものだと思ってください。
「カレーの作り方」という名前をつけておけば、次からは「カレー作って！」と言うだけで済みますよね。
プログラミングでも、よく使う動きは名前をつけてまとめておきましょう。
-->

## 4-1. 【やってみよう 1】クリックで色を変えよう！

1. HTMLにボタンを一つ作る。
2. そのボタンを押すと、自分の名前の色が変わるようにJSを書く。

```javascript
const btn = document.querySelector("#color-btn");
const nameTag = document.querySelector("h1");

btn.addEventListener("click", () => {
  nameTag.style.color = "hotpink";
});
```

**✅ 成功イメージ:** ボタンを押した瞬間に、パッと名前の色が変わったら大成功！

## 5-1. プロの技：クラスを付け外しする

`style` を直接いじるよりも、CSSのクラスを切り替える方が賢いやり方です。

- **`classList.toggle("クラス名")`**:
  - クラスがなければ「つける」。
  - クラスがあれば「はずす」。

```javascript
// クリックするたびに .active クラスが入れ替わる！
btn.addEventListener("click", () => {
  document.body.classList.toggle("dark-mode");
});
```

<!-- 
【講師用台本】
一つレベルアップしたテクニックを紹介します。
JSで直接「色を赤にして、背景を黒にして…」と細かく指定するのは大変ですよね。
そこで、CSSであらかじめ「ダークモードの時はこうする」というセットを作っておきます。
JSの役割は、そのセット（クラス）を「オンにするかオフにするか」を決めるだけ。
これなら、デザインの修正もCSSを直すだけで済むので、すごく効率的なんですよ！
-->

## 5-2. 豆知識：バブリング（泡）現象

イベントは、発生した場所から「親」の方へ泡のように上がっていきます。

- ボタンをクリックすると、その親の `div` も、その親の `body` もクリックされたとみなされます。
- **なぜ知っておくの？**
  - 「ボタンだけ反応させたいのに、後ろのカードまで反応しちゃった！」という時に、この知識が役立ちます。

<!-- 
【講師用台本】
ちょっとマニアックですが、大事な知識。イベントの「バブリング」です。
池に石を投げた時に波紋が広がるように、イベントも発生した場所から外側へ伝わっていきます。
最初は気にしなくて大丈夫ですが、「あれ、思わぬところが反応してるぞ？」と不思議に思ったら、この泡（バブル）のことを思い出してくださいね。

## 6-1. 入力に合わせて動く：`input` イベント

ボタンを押さなくても、キーボードを叩くたびに反応させることができます。

```javascript
const input = document.querySelector("#name-input");
const display = document.querySelector("#greeting");

input.addEventListener("input", (e) => {
  // 「e.target.value」で今入力されている文字が取れる！
  display.textContent = `こんにちは、${e.target.value}さん！`;
});
```

<!-- 
【講師用台本】
最後は「入力」との連動です。
検索フォームなどで、文字を入れるたびに検索結果が変わる仕組みを見たことがありますよね？
あれはこの `input` イベントを使っています。
ユーザーが何を打ち込んだか、リアルタイムで追いかけることができるんです。
自分の名前を打ち込んだら、画面上の挨拶がどんどん書き換わる仕組みを作ってみましょう。
-->
