## 1-1. 標準フローという「不自由」

HTMLは放っておくと、要素を「上から下へ」積み上げます。

- **デフォルトの状態:**
  - `<h1>` も `<p>` も `<div>` も、画面の端まで横幅を占領します。
  - 次の要素は、その下に強制的に送られます。

**💡 悩み:** 
- 「メニューを横に並べたいのに…」
- 「写真を2枚並べたいのに、縦になっちゃう…」

<!-- 
【講師用台本】
HTMLの初期状態は、とても生真面目です。
要素を一つ置くと、その横には誰もいさせず、次の要素を下に押しやります。
これを「標準フロー」と呼びます。
「サイドバーを左に、メインを右に」といったモダンなデザインを作るには、この不自由なルールを打ち破る必要があるんです。
-->

---

## 2-1. Flexbox：最強の「整列」ツール

「1列」の並びを自由自在に操るための仕組みです。

- **親要素（入れ物）:** `display: flex;` を書く。
- **子要素（中身）:** 勝手に横並びになります！

```css
.container {
  display: flex; /* これだけで魔法が始まる */
}
```

**💡 イメージ:** 
- バラバラだった要素が、一列に並んで整列する感じです。

<!-- 
【講師用台本】
そこで登場するのが Flexbox です！
親となる箱に `display: flex;` と一行書くだけで、中に入っている子たちが「はい、整列！」と言わんばかりに横に並び始めます。
メニューバーや、横並びのカードなど、Webサイトのいたるところで使われている超重要テクニックです。
-->

---

## 2-2. どっちに並べる？：`flex-direction`

横並びだけでなく、縦並びに戻すこともできます。

- **`row` (デフォルト):** 横に並べる。
- **`column`:** 縦に並べる。

**なぜ縦にするの？**
- スマホでは縦並び、PCでは横並び、といった切り替えが楽になるからです。
- これについては来週の「レスポンシブ」で詳しくやります！

<!-- 
【講師用台本】
並べる方向も自由自在です。
基本は横並び (`row`) ですが、`column` を指定すると縦に並びます。
「え、縦に戻すなら最初からflex使わなくていいんじゃ？」と思うかもしれませんが、
flexを使うと、縦並びにした時でも「上下の中央寄せ」などがすごく楽にできるというメリットがあるんですよ。
-->

---

## 2-3. 【やってみよう 1】横並びにしてみよう！

1. 先週の「スキルセットのリスト (`ul`)」に `display: flex;` を適用してみる。
2. リストのポッチを消して、ボタンのように並べてみる。

```css
ul {
  display: flex;
  list-style: none; /* ポッチを消す */
  gap: 10px;        /* 隙間をあける */
}
```

**✅ 成功イメージ:** 縦並びだったリストが、スッと横一列に並んだら成功！

## 3-1. 中央寄せの魔法（二大巨頭）

要素を「ど真ん中」に置きたい時の鉄板レシピです。

- **`justify-content`**: 主軸（横方向）の揃え方。
  - `center`: 中央寄せ。
  - `space-between`: 両端に広げて均等配置。
- **`align-items`**: 交差軸（縦方向）の揃え方。
  - `center`: 上下の中央寄せ。

<!-- 
【講師用台本】
CSS最大の難問と言われた「中央寄せ」が、Flexboxならたった2行で解決します。
`justify-content` で横方向、`align-items` で縦方向。
どちらも `center` にすれば、中身は完全にど真ん中に居座ります。
これを覚えるだけで、皆さんのデザインの悩みは8割くらい解消されるはずです！
-->

---

## 3-2. 豆知識：なぜ「Flex」なの？

- **由来:** Flexible（柔軟な）という意味です。
- **柔軟性:** 
  - 中身の量に合わせて、箱の大きさが伸び縮みします。
  - 要素が増えても、崩れずに綺麗に並べてくれる「優等生」です。
  - 昔のレイアウト手法（floatなど）よりも圧倒的に壊れにくいのが特徴です。

<!-- 
【講師用台本】
Flexboxの「Flex」は「柔軟」という意味です。
画面の幅が狭くなったり、中身の文字が少し増えたりしても、いい感じに調整してくれます。
昔のプログラミングは要素が1ピクセルでもずれると全体が崩れるなんてこともありましたが、
今はFlexboxのおかげで、とても安全にレイアウトを組めるようになったんです。
-->

---

## 4-1. CSS Grid：ページを分割する

Flexboxが「1列」なら、Gridは「面（2次元）」を操ります。

- **役割:** ページ全体を「2カラム」や「タイル状」に分割する。
- **基本:**
  - `display: grid;`
  - `grid-template-columns`: 列の数を決める。

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr; /* 左固定、右は残り全部 */
}
```

<!-- 
【講師用台本】
Flexboxで細かいパーツを並べるのに対し、ページ全体の大きな枠組みを作るのに最適なのが「CSS Grid」です。
「左側にサイドバー、右側に広いメインエリア」といった、今から作るダッシュボードの基本構造は、Gridを使うと驚くほど簡単に作れます。
方眼紙に線を引くような感覚で、画面を分割してみましょう。
-->

---

## 5-1. 魔法の単位 `fr` (Fraction)

Gridだけで使える、最強に便利な単位です。

- **意味:** 「分数（比率）」という意味。
- **使い方:**
  - `1fr 1fr 1fr`: 3等分。
  - `1fr 2fr`: 1対2の比率で分ける。
- **メリット:** 
  - `%` と違って、隙間（gap）を引いた後の残り時間を自動で計算してくれます。

<!-- 
【講師用台本】
Gridを使いこなす鍵が、この `fr` という単位です。
「1fr」は「残っているスペースの1個分」という意味。
`1fr 2fr` と書けば、自動で「1：2」の幅に分けてくれます。
自分で「33.333%...」なんて計算する必要はもうありません。Gridにお任せしましょう！
-->

---

## 6-1. 隙間は `gap` にお任せ

Marginで隙間を作る時代の終わりです。

- **`gap: 20px;`**: 
  - FlexboxでもGridでも使えます。
  - **隣り合う要素の間だけに** 隙間を作ってくれます。
  - 一番端っこに変な余白が残らないのが最大のメリット！

<!-- 
【講師用台本】
昔のエンジニアは、要素の間に隙間を作るためにMarginを計算して苦労していました。
でも今は `gap` があります。
親の箱に `gap: 20px` と書くだけで、子供たちの間にだけ綺麗に20pxの隙間が入ります。
一番右端や一番下に余計な隙間が出ないので、デザインが絶対に崩れません。
-->

---

## 6-2. 【やってみよう 2】ページを2分割しよう！

1. `<body>` 直下の大きな入れ物（例: `.wrapper`）に `grid` を適用する。
2. プロフィール部分（サイドバー）とコンテンツ部分を分けてみる。

```css
.wrapper {
  display: grid;
  grid-template-columns: 1fr 3fr; /* 1:3で分割 */
  gap: 30px;
  min-height: 100vh; /* 画面いっぱいの高さにする */
}
```

<!-- 
【講師用台本】
さあ、いよいよダッシュボード化の瞬間です！
ページ全体を包むタグに、Gridをかけてみましょう。
`1fr 3fr` と指定して、左を少し狭く、右を広くしてみてください。
背景色をそれぞれのエリアで変えてみると、ハッキリと「2つのエリア」に分かれたのが分かって、感動しますよ！
-->

## 7. まとめ：ダッシュボードをレイアウトしよう！

これまでの「自己紹介カード」を、本格的な「ダッシュボード」に進化させます！

- **目標:**
  - 全体を「サイドバー」と「メイン」の2カラムにする (Grid)。
  - ナビゲーションメニューを綺麗に並べる (Flexbox)。
  - スキル一覧をタイル状に並べる (Grid)。
  - 常に画面中央に要素を配置する (Flexbox)。

<!-- 
【講師用台本】
今日の総仕上げです！
縦長だったプロフィールページを、横に広がる「ダッシュボード」に改造しましょう。
左側に名前や写真、右側に詳しい情報。
FlexboxとGrid、どちらを使えばいいか迷ったら「一列ならFlex、枠組みならGrid」と思い出してください。
今日一日で、皆さんのページはぐっと「アプリ」らしくなりますよ！
-->

## 8. 迷った時のチェックリスト

1. **親に書いた？:** `display: flex` や `grid` は、「並べたい子要素」の **親** に書きます。
2. **スペル:** `justify-content` など、綴りが長いので注意。
3. **高さ:** 上下中央に寄らない時は、親要素に `height`（高さ）があるか確認。
4. **frの計算:** 合計でいくつになるかイメージしてみよう。
