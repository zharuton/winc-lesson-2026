## 1-1. 「同期」と「非同期」の違い

インターネットの通信は、時間がかかることがあります。

- **同期処理（これまでの書き方）:**
  - 1つの仕事が終わるまで、次の行に進まない。
  - **問題:** 通信が遅いと、画面がフリーズして動かなくなる！
- **非同期処理（これからの書き方）:**
  - 「データの準備ができたら教えてね！」と頼んでおいて、次の仕事を進める。
  - **メリット:** 読み込み中も、他のボタンを押したりスクロールしたりできる！

<!-- 
【講師用台本】
プログラミングの世界には「同期」と「非同期」という言葉があります。
同期は、レジに行列ができるお店のようなもの。前の人の会計が終わるまで、後ろの人は一歩も動けません。
非同期は、フードコートの呼び出しベルのようなもの。注文したらベルをもらって、席で自由に待っていられますよね。
通信には時間がかかるので、ユーザーを待たせてイライラさせないために、この「呼び出しベル（非同期）」の考え方が不可欠なんです。
-->

## 2-1. API：データの「自動販売機」

他のサービスが持っているデータを使わせてもらう窓口のことです。

- **API (Application Programming Interface)**:
  - 自分のサイト 「GitHubさん、私の情報をください！」
  - GitHub 「はい、どうぞ！（データを返す）」
- **例:**
  - 天気予報 API / ニュース API / ポケモン API

<!-- 
【講師用台本】
「API」と聞くと難しそうですが、要は「情報の窓口」です。
例えば、自分のサイトで明日の天気を表示したい時、わざわざ自分で気象観測をする必要はありません。
気象庁などのAPIに向かって「天気を教えて！」とリクエストを送れば、データを送り返してくれます。
世界中の便利なサービスが、開発者向けにこの「窓口」を開放してくれているんですよ。
-->

## 3-1. JSON：世界共通の「データの形」

APIから返ってくるデータは、ほとんどがこの **JSON** という形式です。

- **JSON (JavaScript Object Notation)**:
```json
{
  "name": "WINCくん",
  "followers": 150,
  "isStudent": true
}
```

- **特徴:** 
  - JavaScriptのオブジェクトにそっくり。
  - とても軽くて、人間にも読みやすい。

<!-- 
【講師用台本】
APIとやり取りする時の共通言語が「JSON（ジェイソン）」です。
ホラー映画のキャラクターじゃありませんよ（笑）。
見ての通り、中身はとてもシンプルです。
JavaScriptの変数にすごく似ているので、私たちWebエンジニアにとっては非常に扱いやすいデータの形式なんです。
-->

## 4-1. `fetch`：データを取りに行く命令

ブラウザに備わっている、お使い（通信）専用の命令です。

```javascript
// 「このURLにデータを取りに行って！」
fetch("https://api.example.com/data")
```

**💡 ポイント:** 
- `fetch` は非同期処理なので、すぐに結果が返ってきません。
- 「お使いに行ってくるから、後で連絡するね！」という **Promise（約束）** が返されます。

<!-- 
【講師用台本】
実際にデータを取りに行くには、`fetch（フェッチ）` を使います。
「取ってくる」という意味ですね。
これを実行すると、ブラウザが裏側でパタパタとお使いに行ってくれます。
ただ、お使いからいつ帰ってくるかは分かりませんよね。
そこで、「Promise（約束）」という仕組みを使って、帰ってきた後の処理を予約しておくことになります。
-->

---

## 5-1. スッキリ書ける：`async` / `await`

Promise（予約）の処理を、まるで普通のプログラムのように読みやすく書く魔法です。

```javascript
async function getProfile() {
  // お使いが帰ってくるまで、ここで「待機(await)」する
  const response = await fetch("https://api.github.com/users/winc-titech");
  
  // 文字列をJSON（オブジェクト）に変換するのを「待機」
  const data = await response.json();
  
  console.log(data.name); // 取得したデータの中身を表示！
}
```

<!-- 
【講師用台本】
昔のJavaScriptは非同期の処理を書くのがすごく大変だったんですが、今は `async` と `await` があります。
関数の前に `async` をつけて、時間のかかる処理の前に `await` と書くだけ。
「ここでお使いが帰るのを待つよ！」という意味になります。
すごく直感的で読みやすいですよね。今のプロの現場でも、この書き方が一番人気です。
-->

## 6-1. 【やってみよう 1】犬の写真を召喚しよう！

1. 公開されている「ランダムな犬の画像API」を叩いてみる。
2. 取得したURLを `img` タグの `src` にセットする。

```javascript
async function getDog() {
  const res = await fetch("https://dog.ceo/api/breeds/image/random");
  const data = await res.json();
  
  const img = document.querySelector("img");
  img.src = data.message; // 犬の画像のURLがここに！
}
```

**✅ 成功イメージ:** 読み込むたびに、違うワンちゃんが表示されたら大成功！

## 6-2. 失敗に備える：`try...catch` の優しさ

インターネットは、時々繋がらないことがあります。

```javascript
async function getData() {
  try {
    const res = await fetch("...");
    // 成功した時の処理
  } catch (error) {
    // 失敗（ネット切れなど）した時の処理
    console.log("お使いに失敗しました...", error);
    alert("データを取ってこれませんでした。");
  }
}
```

<!-- 
【講師用台本】
プロのコードには「優しさ」が必要です。
地下鉄にいて電波が切れたり、相手のサーバーがダウンしていることもあります。
そんな時、プログラムがエラーで止まってしまうのではなく、「ごめんね、今はデータが取れないんだ」と優しく教えてあげる。
そのために `try...catch` という構文を使って、失敗した時の保険をかけておきましょう。
-->

## 8. 迷った時のチェックリスト

1. **URLのスペル:** 1文字でも違うと、お使いは目的地に着けません。
2. **`await` 忘れ:** `await` を忘れると、お使いが帰る前に次の行に進んでしまいます。
3. **`json()`:** 返ってきたままだとただの「文字列」です。必ずオブジェクトに変換して！
4. **console.log:** 詰まったら、まず `data` の中身をログに出して観察しましょう。

<!-- 
【講師用台本】
APIの連携で詰まったら、ここを見てください。

「今、どんな形のデータが返ってきているのかな？」を常に観察するのが、APIマスターへの一番の近道ですよ。
