## 1-1. Next.js：最強の「部品組み立て」ツール

これまでの開発と、これからの開発は何が違うのでしょうか？

- **これまで (Vanilla):** 
  - 1枚の長い長いHTMLを頑張って書く。
  - JSで「h1を探して色を変えて…」と細かく指示を出す。
- **これから (Next.js):**
  - ボタン、カード、メニューなどを「部品（コンポーネント）」として作る。
  - 部品をレゴブロックのように組み合わせて、ページを作る！

<!-- 
【講師用台本】
Next.jsを使う一番の理由は「部品化」ができることです。
例えば「おしゃれなボタン」を一個作っておけば、あとはそれを呼び出すだけで何度でも使い回せます。
修正する時も、その部品一個を直せば全部のページに反映される。
この「レゴブロック」のような作り方が、今のWeb開発の主流なんです。
-->

## 2-1. プロジェクトの中身を覗いてみよう

`create-next-app` というコマンドを打つと、たくさんのファイルが現れます。

- **`app/`**: ここが皆さんの「メイン作業場」です。
- **`app/page.tsx`**: ブラウザに表示される「中身」そのもの。
- **`app/layout.tsx`**: ページ全体の「共通の枠組み（ヘッダーとか）」。
- **`public/`**: 画像やアイコンを入れる場所。

<!-- 
【講師用台本】
Next.jsのプロジェクトを作ると、最初はファイルの多さにびっくりするかもしれません。
でも、今日触るのは `app` フォルダの中だけです。
「page」が中身、「layout」が外枠、と覚えておけばOK！
フォルダ構成が最初から決まっているので、迷わず開発に集中できるのがNext.jsの良いところですね。
-->

## 3-1. JSX：HTMLが進化した姿

Next.js（React）では、JavaScriptの中に直接HTMLのようなものを書きます。これを **JSX** と呼びます。

- **微妙に違うルールに注意！**
  - `class` → `className` と書く。
  - 閉じタグは絶対に必要！ (例: `<img />`, `<br />`)
  - `{ }` を使えば、HTMLの中でJSの変数が使える。

<!-- 
【講師用台本】
新しい書き方「JSX（ジェーエスエックス）」を覚えましょう。
見た目はほとんどHTMLですが、中身はJavaScriptです。
一番間違いやすいのが `class` を `className` と書かなきゃいけないこと。
最初は「あれ？」と思うかもしれませんが、これによってJavaScriptのパワーをそのままHTMLに持ち込めるようになる、すごい発明なんですよ。
-->

---

## 4-1. Tailwind CSS：クラス名だけでデザイン

CSSファイルを作らず、HTMLタグに直接「意味のある名前」を書いてデザインします。

```tsx
// 青い背景、白い文字、角を丸く、余白を4
<button className="bg-blue-500 text-white rounded-lg p-4">
  ボタン
</button>
```

- **メリット:** 
  - CSSファイルを往復しなくていい。
  - クラス名に悩まなくていい。

<!-- 
【講師用台本】
ここで最強のデザインツール「Tailwind CSS」を紹介します。
今までCSSファイルに書いていた指示を、タグの `className` に直接書き込んでいきます。
`bg-blue-500` は「青い背景」、`p-4` は「パディング4」といった具合です。
「え、覚えるのが大変そう！」と思うかもしれませんが、これに慣れると爆速でページが出来上がっていくので、もう手放せなくなりますよ！
-->

---

## 5-1. レイアウト：みんなの「外枠」

すべてのページで共通のパーツ（ヘッダーやサイドバー）は、`layout.tsx` に書きます。

- **役割:** 
  - ページを移動しても、サイドバーが消えないようにする。
  - 全体の背景色などを一括で決める。

```tsx
export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <nav>サイドバー</nav>
        {children} {/* ここに各ページの中身が入る！ */}
      </body>
    </html>
  );
}
```

<!-- 
【講師用台本】
`layout.tsx` は、ページの「入れ物」です。
中身が入れ替わっても、外側のヘッダーやフッターはそのまま残したいですよね。
その共通部分をここに書きます。
`{children}` という不思議な言葉がありますが、ここに各ページ（page.tsx）の中身がスポッと収まるイメージです。
-->

## 6-1. 【やってみよう 1】最初のページを書き換えよう！

1. `app/page.tsx` の中身を全部消す。
2. 自分の名前を表示するだけのシンプルなコードに書き換える。

```tsx
export default function Home() {
  return (
    <main className="p-10">
      <h1 className="text-4xl font-bold text-orange-500">
        Next.js の世界へようこそ！
      </h1>
      <p>これは私の最初のモダンWebページです。</p>
    </main>
  );
}
```

**✅ 成功イメージ:** ブラウザが自動で更新されて、新しい文字が出たら成功！

<!-- 
【講師用台本】
それでは最初の実践です！
デフォルトで入っているNext.jsの説明を全部消して、自分専用のページに書き換えましょう。
保存した瞬間に、ブラウザがリロードしなくてもパッと切り替わるはずです。
この快適な開発体験こそが、モダン開発の醍醐味です！
-->

## 7. まとめ：ダッシュボードをお引越し！

これまで作ってきたHTMLの中身を、`app/page.tsx` にコピーして、モダンな書き方に直しましょう。

- **目標:**
  - HTMLタグをJSX（`className`など）に書き換える。
  - Tailwind CSS を使って、これまでのデザインを再現する。
  - 画像は `public/` フォルダに入れて読み込む。
  - コンパイルエラー（赤い画面）が出ないように閉じタグをチェック！

## 8. 迷った時のチェックリスト

1. **`className`:** `class` と書いていませんか？
2. **閉じタグ:** `<img>` や `<br>` は `<img />` や `<br />` になっていますか？
3. **リターン:** `return ( ... );` のカッコの中にHTMLが入っていますか？
4. **小文字:** HTMLタグ（`div`, `p`など）は小文字で書き始めていますか？

<!-- 
【講師用台本】
Next.jsでエラーが出た時にチェックしてほしいポイントです。
特に閉じタグ忘れは、JSXでは致命的なエラーになります。
ブラウザに表示されるエラーメッセージをよく読むと、「〇行目のタグが閉じてないよ！」と親切に教えてくれますよ。
-->
