## 1-1. なぜレスポンシブが必要なの？

PCで作ったサイトをそのままスマホで見ると、いろいろ困ったことが起きます。

- **文字が豆粒:** PC用の幅を無理やりスマホに詰め込むから。
- **横スクロール:** 画像が画面からはみ出してしまう。
- **ボタンが押せない:** 指で押すには小さすぎる。

**💡 解決策:** 
- 「デバイスの大きさに合わせて、デザインを柔軟に変える（レスポンシブ）」こと！

<!-- 
【講師用台本】
Webサイトが誕生した頃は、みんな四角いPCの画面で見ていました。
でも今は、巨大なモニターから、小さなスマホ、タブレットまで様々です。
それぞれのデバイスに合わせて「別のサイト」を作るのは大変ですよね。
そこで、一つのHTMLを「CSSの力」だけで変化させるのがレスポンシブデザインの考え方です。
-->

---

## 2-1. 必須！Viewport（ビューポート）の設定

これがないと、どんなにCSSを頑張ってもスマホ対応できません。

- **おまじないの1行（HTMLのhead内に書く）:**
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

- **意味:**
  - `width=device-width`: 「デバイスの横幅」をそのままサイトの幅として使ってね。
  - `initial-scale=1.0`: 最初はズームせず「1倍」で表示してね。

<!-- 
【講師用台本】
まず一番最初にやるべきこと。HTMLのheadタグの中に、この「ビューポート」の設定を書き込みます。
これがないと、スマホのブラウザは「PC用の広い画面を無理やり縮小して」見せようとしてしまいます。
この1行を書くことで、初めてスマホが「あ、このサイトはスマホの幅に合わせて作ってあるんだな」と理解してくれるんです。
まさにスマホ対応の「合言葉」ですね！
-->

---

## 3-1. スマホを持ってなくても大丈夫！

ブラウザの「デベロッパーツール」を使えば、スマホ画面をシミュレーションできます。

1. ブラウザ（Chromeなど）で **F12**（または右クリック→検証）を押す。
2. 左上にある **「デバイス切り替えボタン」** をクリック。
3. 画面の端をマウスでドラッグして、幅を変えてみる。

**💡 ポイント:** 
- iPhoneやPixelなど、特定の機種での見え方も選べます！

<!-- 
【講師用台本】
「自分のスマホで確認しなきゃいけないの？」と思った方、安心してください。
開発者はみんな、PCのブラウザについている「デベロッパーツール」を使って確認しています。
画面の横幅をグーッと縮めていくと、デザインが崩れる瞬間が見つかるはずです。
その「崩れた瞬間」こそが、デザインを切り替えるべきタイミングなんですよ。
-->

---

## 4-1. メディアクエリ：デザインの切り替えスイッチ

「もし画面の幅が〇〇px以下なら、この設定にして！」という命令です。

```css
/* PC用の基本設定 */
.container { display: flex; }

/* スマホ用（幅が 768px 以下の時）の切り替え設定 */
@media (max-width: 768px) {
  .container {
    flex-direction: column; /* 縦並びにする！ */
  }
}
```

<!-- 
【講師用台本】
今日のメインディッシュ、「メディアクエリ」です。
`@media` という特別な書き方の中に、スマホ専用のCSSを書きます。
例えば、「PCでは横並びだけど、幅が768pxより小さくなったら縦並びに変えてね」といった指示が出せます。
このスイッチを使いこなせば、一つのサイトでPC用・スマホ用の二つの顔を持たせることができるんです。
-->

---

## 5-1. モバイルファースト：賢い書き方

「PC版を作ってからスマホ版を作る」のではなく、その逆が今のトレンドです。

1. **まず基本（外側）に「スマホ用」の設定を書く。**
2. **`@media (min-width: 768px)` の中に「PC用」の設定を書く。**

**なぜ？**
- スマホの方が画面がシンプルなので、コードが短くなる。
- 外出先（スマホ）で見る時に、余計なコードを読み込まなくて済む。

<!-- 
【講師用台本】
これ、実はすごく賢いやり方なんです。
スマホは画面が小さくてシンプルですよね？ だからまず「スマホ用」をベースに書く。
そして、画面が広くなった時だけ「PC用の複雑なレイアウト（2カラムなど）」を追加してあげる。
これを「モバイルファースト」と呼びます。
プロの現場では、この書き方が標準になっていることが多いんですよ。
-->

---

## 5-2. 豆知識：ブレイクポイントって？

デザインが切り替わる「境目」のことです。

- **よく使われる数字:**
  - `768px`: タブレットとPCの境目。
  - `480px`: スマホとタブレットの境目。

**💡 ヒント:** 
- 厳密な数字にこだわらなくてもOK！
- 自分のデザインが「あ、崩れたな」と思った幅を境目にするのが一番正解に近いです。

<!-- 
【講師用台本】
切り替える幅のことを「ブレイクポイント」と言います。
昔はiPhoneの幅に合わせて…なんて決めていましたが、今はデバイスの種類が多すぎてキリがありません。
なので、最近は「自分のサイトが崩れて見苦しくなった幅」をブレイクポイントにするのが一般的です。
「実機に合わせるのではなく、コンテンツに合わせる」のがコツですね。
-->

## 6-1. 画像のはみ出しを防ぐ「魔法のCSS」

画像が画面からはみ出して横スクロールが出るのを防ぎます。

```css
img {
  max-width: 100%; /* 親の幅より大きくならない */
  height: auto;    /* 縦横比を保つ */
}
```

**💡 これだけで:** 
- どんな大きな画像も、画面の幅に収まってくれます！

<!-- 
【講師用台本】
レスポンシブ対応で一番多いトラブルが「画像がはみ出すこと」です。
これを防ぐために、CSSの最初にこの2行を書いておきましょう。
「親の箱（画面）からはみ出さないでね、でも形は崩さないでね」という命令です。
これだけで、スマホ対応の苦労が半分くらい減りますよ（笑）。
-->

---

## 6-2. 【やってみよう 1】画面幅で背景色を変えてみよう！

1. メディアクエリを書いて、幅を縮めた時に背景色が変わるか試してみる。

```css
body { background-color: white; }

@media (max-width: 600px) {
  body { background-color: lightblue; }
}
```

**✅ 成功イメージ:** 幅を狭くした瞬間、パッと色が変わったらメディアクエリ成功！

<!-- 
【講師用台本】
まずは肩慣らしです。
デベロッパーツールで幅を縮めてみてください。
ある幅を境に背景色がパッと変わったら、メディアクエリの魔法が発動した証拠です。
「お、変わった！」という瞬間をぜひ体験してください！
-->

## 7. まとめ：ダッシュボードをスマホ対応させよう！

先週作った「2カラム（サイドバー＋メイン）」を、スマホでは「縦1列」にします！

- **目標:**
  - Grid の `grid-template-columns` を、スマホでは `1fr` にする。
  - プロフィール写真をスマホでは中央に寄せる。
  - 余白 (`padding`) をスマホ用に少し狭くして、画面を有効活用する。

<!-- 
【講師用台本】
今日の総仕上げです！
PCでは左にプロフィール、右に情報が表示されていました。
それをスマホで見た時は、プロフィールが上に、情報がその下に、というふうに縦に並ぶように書き換えてみましょう。
Gridの列の設定を `1fr` に変えるだけ。たったそれだけで、あなたのダッシュボードは「スマホアプリ」のような使い心地になります！
-->

## 8. 迷った時のチェックリスト

1. **ViewPort:** HTMLのheadに `<meta>` タグはありますか？
2. **カッコ `{ }`:** `@media { ... }` の中に、さらに `セレクタ { ... }` が入っています。カッコの数に注意！
3. **キャッシュ:** 変更が反映されない時は、ブラウザを「強力に再読み込み」してみよう。
4. **はみ出し:** 横スクロールが出ていないかチェック！

<!-- 
【講師用台本】
