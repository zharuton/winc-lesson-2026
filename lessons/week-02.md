## 1-1. CSSってどんな役割？

HTMLが「中身」なら、CSSは「見た目」を担当します。

- **Cascading Style Sheets** の略。
- **役割:**
  - **装飾:** 文字の色を変える、背景に画像を貼る。
  - **レイアウト:** 大きさを決める、横に並べる。
  - **アニメーション:** フワッと動かす。

**💡 分業が大事:** 
- HTMLは「文章の構造」に集中。
- CSSは「デザイン」に集中。

<!-- 
【講師用台本】
CSSは、Webサイトの「お化粧」や「ファッション」のようなものです。
HTMLで「ここに名前があるよ」「ここに写真があるよ」という情報を整理して、
CSSで「名前は大きくオレンジ色にして、写真は角を丸くしよう」といった指示を出します。
役割をしっかり分けることで、後からデザインだけを変えたい時もすごく楽になるんですよ。
-->

---

## 1-2. どこに書くのが正解？

CSSを書く場所は3つありますが、**「外部ファイル」**がプロの標準です！

1. **外部ファイル（おすすめ！）**: `style.css` を作って読み込む。
2. **headタグ内**: `<style>` タグの中に書く。
3. **HTMLタグに直接**: `style="..."` と書く（おすすめしません）。

**なぜ外部ファイルなの？**
- HTMLがスッキリして読みやすくなるから。
- 複数のページで同じデザインを使い回せるから。

<!-- 
【講師用台本】
CSSをどこに書くか、という問題。
実はHTMLファイルの中に直接書くこともできるんですが、サークルでは「外部ファイル」を推奨します。
別のファイルに分けることで、HTMLは「文章」、CSSは「デザイン」とスッキリ整理できます。
部屋の片付けと同じで、役割ごとに場所を分けると、どこに何があるかすぐ分かるようになりますよ！
-->

## 1-3. 【やってみよう 1】CSSを繋いでみよう！

1. 先週のフォルダに `style.css` というファイルを新しく作る。
2. `index.html` の `<head>` 内に下の1行を書く。

```html
<link rel="stylesheet" href="style.css">
```

3. `style.css` にこれだけ書いて保存！
```css
h1 { color: red; }
```

**✅ 成功イメージ:** 自分の名前が「赤色」に変わったら合流成功です！

## 2-1. CSSの文法：3つのキーワード

CSSは、たった3つの言葉の組み合わせでできています。

- **セレクタ:** 「誰に」（例: `h1`）
- **プロパティ:** 「何を」（例: `color`）
- **値:** 「どうする」（例: `blue`）

```css
h1 {
  color: blue;
  font-size: 24px;
}
```

**💡 ポイント:** 
- `{ }` でプロパティを囲みます。
- 最後に `;` (セミコロン) を忘れずに！これがないと動きません。

<!-- 
【講師用台本】
CSSの書き方は、実はすごくシンプルです。
「誰に」「何を」「どうする」の3点を伝えるだけ。
h1（誰に）、color（何を）、blue（どうする）。
よくある間違いは、最後のセミコロンを忘れてしまうこと。
日本語の「。」と同じで、ここで命令が終わりだよ！という意味なので、忘れないであげてくださいね。
-->

---

## 2-2. セレクタの指定方法：タグ・クラス・ID

「誰に」を指定する方法はいろいろあります。

- **タグ指定 (`h1`, `p` など)**:
  - そのタグ全部に適用されます。
- **クラス指定 (`.name`, `.card` など)**: 
  - 名前（クラス名）をつけた要素だけに適用されます。
  - **一番よく使います！** 先頭に `.` をつけるのがルール。
- **ID指定 (`#main` など)**:
  - そのページで1回しか使えない特別な名前。先頭に `#` をつける。

<!-- 
【講師用台本】
「誰に」を指定するセレクタ。
タグで指定すると、ページ内の全てのpタグが同じ色になってしまいます。
「この段落だけ色を変えたい！」という時は、HTMLの方で `class="highlight"` のように名前をつけて、CSSで `.highlight` と指定します。
クラス名は自分の好きな名前でOKですが、誰が見ても分かる名前にするのがカッコいいエンジニアへの第一歩です！
-->

---

## 2-3. 豆知識：どっちが優先されるの？

もし同じ要素に違う色が指定されたら、どうなるでしょうか？

- **ルール 1: 強いセレクタが勝つ！**
  - `タグ` ＜ `クラス` ＜ `ID` の順で強くなります。
- **ルール 2: 後から書いた方が勝つ！**
  - 強さが同じなら、ファイルの「下」に書いた方が上書きします。

**💡 ヒント:** 
- 迷ったら「クラス」で指定するのがプロのバランス感覚です。

<!-- 
【講師用台本】
CSSには「優先順位」という面白いルールがあります。
もしh1に対して「赤くしろ」という命令と「青くしろ」という命令がぶつかったら、
より具体的な指定（クラス名など）が勝ったり、後から言われたことが優先されたりします。
「あれ？色が変わらないぞ？」と思ったら、どこかで上書きされていないかチェックしてみましょう！
-->

## 3-1. 色と背景を操る

ページ全体の印象は「色」で決まります。

- **文字の色:** `color: #ea580c;`
- **背景の色:** `background-color: #fff7ed;`
- **色の指定方法:**
  - `red`, `blue` などの名前。
  - `#ffffff` などの16進数（プロが一番使うやつ）。
  - `rgb(255, 0, 0)` などの数字。

<!-- 
【講師用台本】
次は色の設定です。
背景色 (`background-color`) を変えるだけで、サイトの雰囲気がガラッと変わります。
色は「red」のように英語でも書けますが、細かなニュアンスを出すためには「#」から始まる6文字のコードを使うのが一般的です。
「好きな色のカラーコード」で検索すると、たくさん素敵な色が出てきますよ！

## 4-1. 文字のデザイン（フォント）

フォントが変わると、サイトの「性格」が変わります。

- **サイズ:** `font-size: 16px;`
- **太さ:** `font-weight: bold;`
- **種類:** `font-family: sans-serif;`
  - `serif`: 明朝体（大人っぽい、高級感）
  - `sans-serif`: ゴシック体（親しみやすい、モダン）

<!-- 
【講師用台本】
文字のデザインです。
大きさ (`font-size`) だけでなく、太さや種類 (`font-family`) も重要です。
カチッとした真面目なサイトにしたいか、可愛らしいふわふわしたサイトにしたいか。
目指すイメージに合わせて選んでみましょう。
ちなみに、Webフォントという技術を使えば、世界中の素敵なフォントを自分のサイトで使うこともできるんですよ。
-->

## 5-1. 超重要：ボックスモデル

CSSを学ぶ上で、避けては通れない「一番大事な考え方」です。

- **すべての要素は「箱」である！**
- 見えない箱の構成:
  - **Content**: 文字や画像そのもの
  - **Padding**: 内側の余白
  - **Border**: 枠線
  - **Margin**: 外側の余白（他の箱との距離）

<!-- 
【講師用台本】
さて、今日一番大切なお話です。「ボックスモデル」。
Webサイトに表示されている文字も画像も、実はすべて透明な「箱」の中に入っています。
この箱をどう調整するかが、CSSのレイアウトの全てだと言っても過言ではありません。
中身、内側の余白、枠線、外側の余白。この4つの層をイメージできるようになりましょう！
-->

---

## 5-2. どっちの余白？ Margin vs Padding

初心者が一番迷うポイントです。

- **Padding (内側):** 
  - 箱の中を広げたい時に使います。
  - 背景色を塗っている場合、その色がついたまま広がります。
- **Margin (外側):** 
  - 隣の要素と距離を置きたい時に使います。
  - 箱の外側なので、背景色はつきません。

<!-- 
【講師用台本】
「余白をあけたいな」と思った時、Marginを使うかPaddingを使うか。
箱の内側に余裕を持たせたいならPadding。
箱そのものを隣から遠ざけたいならMarginです。
例えば、お弁当箱の中でおかず同士を離すのがPadding、お弁当箱と水筒の距離をあけるのがMargin、というイメージですね。
-->

---

## 5-3. 【やってみよう 3】箱を意識してみよう

1. 自己紹介のセクションに背景色と枠線をつけてみる。
2. Padding を変えて、中身の窮屈さを解消してみる。
3. Margin を変えて、他の要素との間隔を調整してみる。

```css
.card {
  background-color: white;
  border: 2px solid #ea580c;
  padding: 20px;
  margin: 10px;
}
```

<!-- 
【講師用台本】
実際に箱をいじってみましょう！
ボーダー (`border`) をつけると、今自分がどのくらいの大きさの箱を扱っているかがハッキリわかります。
Paddingがないと文字が枠線にピタッとくっついて窮屈ですが、少し数値を入れるだけで、ゆったりとした読みやすいデザインになりますよ。
-->

## 6-1. もっとおしゃれにする小技

今のトレンドを取り入れて、もっと垢抜けさせましょう！

- **角を丸くする:** `border-radius: 8px;`
  - 四角い箱を少し丸くするだけで、柔らかい印象になります。
- **影をつける:** `box-shadow: 2px 2px 10px rgba(0,0,0,0.1);`
  - ページから浮き上がっているような立体感が出ます。
- **中央に寄せる:** `margin: 0 auto;`
  - 箱を真ん中に置きたい時の魔法の合言葉です。

<!-- 
【講師用台本】
ちょっとした小技を紹介します。
今のWebデザインでは、カチカチの四角形よりも、少し角を丸くしたデザインが好まれます。
また、薄い影をつけると、カードが浮いているように見えてリッチな感じになります。
これらを組み合わせるだけで、一気に「今どき」のWebサイトになりますよ！
-->
