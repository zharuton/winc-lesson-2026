## 1-1. State：アプリが覚えている「今」

アプリが管理している「変化するデータ」をState（ステート）と呼びます。

- **例:**
  - カウンターの数字。
  - ダークモードがONかOFFか。
  - 入力欄に打ち込んでいる途中の文字。
- **Vanilla JSとの違い:**
  - **Vanilla:** 自分で「h1を探して、文字を書き換えて…」と指示。
  - **React:** 「State（記憶）を変える」だけ。見た目はReactが勝手に直してくれる！

<!-- 
【講師用台本】
プログラミングをしていて、「今のスコアをどこかに保存しておきたいな」と思ったことはありませんか？
それを実現するのが State です。
Reactの凄いところは、私たちが「記憶（データ）を書き換えた」瞬間に、それを画面に反映させる作業をすべて自動でやってくれることです。
私たちはデータの管理だけに集中すればいい。これがReactが世界中で愛されている理由なんです。
-->

## 2-1. `useState`：記憶を司るフック

ReactでStateを使うための標準的な道具です。

```tsx
import { useState } from 'react';

function LikeButton() {
  // [今の値, 書き換えるための関数] = useState(最初の値);
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      👍 {count}
    </button>
  );
}
```

<!-- 
【講師用台本】
具体的な書き方を見てみましょう。`useState` という関数を使います。
これを使うと、「今の値」と「その値を変更するための専用関数」の2つがセットで手に入ります。
おもしろいのは、値を直接書き換えてはいけない、というルールです。
必ず `setCount` のような専用関数を通すことで、Reactに「あ、記憶が変わったから画面を更新しなきゃ！」と気づかせてあげるわけですね。
-->

## 3-1. 必須！ 'use client'

Next.js (App Router) では、デフォルトでは「動き」が制限されています。

- **ルール:**
  - `useState` やボタンのクリック（イベント）を使うファイルには、**一番上に** `'use client';` と書く必要があります。
- **意味:**
  - 「この部品はブラウザの上で動く、インタラクティブなやつだよ！」という宣言です。

<!-- 
【講師用台本】
Next.jsを使う上で、絶対に忘れてはいけない「おまじない」があります。
ファイルの1行目に `'use client';` と書くこと。
Next.jsは、できるだけサーバー側で仕事を済ませようとする効率的なツールですが、
ボタンを押したり State を変えたりするのはブラウザ（クライアント）の仕事。
「これはブラウザで動かしてね！」と教えてあげる必要があるんです。
-->

## 4-1. 【やってみよう 1】拍手ボタンを作ろう！

1. `components/ClapButton.tsx` を作る。
2. `'use client';` を忘れずにつける。
3. 押すたびに数字が増えるボタンにする。

```tsx
'use client';
import { useState } from 'react';

export function ClapButton() {
  const [claps, setClaps] = useState(0);
  return (
    <button onClick={() => setClaps(claps + 1)}>
      👏 {claps}
    </button>
  );
}
```

**✅ 成功イメージ:** ボタンを押すたびに、数字がカウントアップされたら大成功！

## 5-1. 入力との連動（Controlled Component）

入力欄の文字をStateで管理する、プロの定番パターンです。

```tsx
const [text, setText] = useState("");

return (
  <div>
    <input 
      type="text" 
      value={text} 
      onChange={(e) => setText(e.target.value)} 
    />
    <p>入力中：{text}</p>
  </div>
);
```

**💡 ポイント:** 
- `value` と `onChange` をセットで使うことで、Stateと入力欄が「完全同期」します。

<!-- 
【講師用台本】
フォームの入力もStateの得意分野です。
ユーザーが1文字打つたびに `onChange` が発動し、Stateの中身を最新の状態に書き換えます。
その最新のStateをまた入力欄の `value` に戻す。
これで、プログラムが常に入力内容を完璧に把握している状態になります。
検索フォームや、コメント欄など、あらゆる場所で使われるテクニックですよ。
-->

## 6-1. 豆知識：React の「賢い」更新

なぜReactは爆速で画面を書き換えられるのでしょうか？

- **仮想DOM (Virtual DOM):**
  - Reactは、頭の中に「理想のHTMLのコピー」を持っています。
  - Stateが変わると、新しい理想のコピーを作ります。
  - **「前のコピー」と「今のコピー」を比較して、変わった場所だけ** を実際のブラウザに反映させます。
- **メリット:** 全部を書き直さないから、すごくスムーズに動く！

<!-- 
【講師用台本】
Reactが裏側で何をやっているのか。
実は、画面全体を書き直しているわけではありません。
「変わったところだけを見つけて、そこだけをピンポイントで直す」という、職人技のようなことを一瞬でやっています。
この仕組みのおかげで、私たちは複雑なことを考えずに「データを変えるだけ」で済んでいるんですね。
-->

## 8. 迷った時のチェックリスト

1. **'use client':** ファイルの一番上に書いていますか？
2. **import:** `import { useState } from 'react';` はありますか？
3. **専用関数:** `count = count + 1` のように直接代入していませんか？（必ず `setCount` を使おう！）
4. **イベント名:** `onclick` ではなく `onClick`（Cは大文字）ですか？

<!-- 
【講師用台本】
うまくいかない時はここをチェック。
特によくあるのが、Stateを直接書き換えてしまうミス。
Reactの世界では「専用の関数を使って、丁寧にお願いする」のがルールです。
また、HTMLでは小文字だった `onclick` が、Reactでは `onClick` に変わるという、ちょっとした罠にも注意してくださいね。
-->
