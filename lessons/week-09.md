## 1-1. Git：エンジニアの「タイムマシン」

コードの変更履歴を、完璧に記録しておくための道具です。

- **バックアップ:** 「あ、バグった！」という時、いつでも過去に戻れる。
- **履歴:** 「いつ、何を、なぜ変えたか」が全部わかる。
- **チーム開発:** 複数人で同じファイルを直しても、上手く合流させてくれる。

**💡 イメージ:** 
- ゲームの「セーブポイント」を、好きな時に何度でも作れる感じです。

<!-- 
【講師用台本】
「昨日は動いてたのに、今日いじったら動かなくなった！しかもどこを直したか忘れた！」
そんな絶望から救ってくれるのが Git です。
ファイルを上書き保存するのではなく、「この時の状態をセーブ！」と記録を残していくことができます。
失敗を恐れずに新しいことに挑戦できるようになる、エンジニアにとって最強の味方なんですよ。
-->

---

## 2-1. GitHub：エンジニアの「SNS」

Gitで作った「記録」を、インターネット上に保存しておく場所です。

- **役割:**
  - 自分のコードを世界に公開できる。
  - 他人のコードを見て勉強できる。
  - プロフィール代わりになり、就職やインターンでも使われる。

**💡 豆知識:** 
- 世界中で数千万人のエンジニアが使っている、開発者の「聖地」です。

<!-- 
【講師用台本】
Gitが「手元のタイムマシン」なら、GitHubは「ネット上の博物館」です。
自分の作った素晴らしいコードを展示したり、世界中の凄いエンジニアのコードを覗き見したりできます。
GitHubを使いこなせている＝立派な開発者の仲間入り、と言っても過言ではありません！
-->

## 3-1. 3つの合言葉を覚えよう

Gitでコードを保存・公開する手順は、いつもこの3ステップです。

1. **`git add .`**: 「セーブするファイル」を選ぶ（準備）。
2. **`git commit -m "メッセージ"`**: セーブを実行！「何をしたか」メモを残す。
3. **`git push`**: セーブデータをGitHubに送り届ける。

<!-- 
【講師用台本】
Gitの操作は、呪文のように決まった手順があります。
まず `add` で「これを保存したいよ！」と選びます。
次に `commit` で「〇〇を修正したよ」というメッセージを添えてセーブ。
最後に `push` で、GitHubという雲の上のサーバーにデータを転送します。
この「add, commit, push」は、エンジニアが毎日100回は唱える魔法の言葉です。
-->

## 4-1. デプロイ：世界デビューの瞬間

「自分のPCの中」にあるページを「インターネット上」に置くことです。

- **ローカル:** 自分しか見られない（自分の家の中）。
- **デプロイ後:** 誰でもURLさえあれば見られる（街の広場）。
- **デプロイに必要なもの:** 「ホスティングサービス」という、サイトを預かってくれるサーバー。

<!-- 
【講師用台本】
「デプロイ」という言葉を覚えましょう。「配置する」という意味です。
今まで私たちは自分のPC（ローカル）で作業してきましたが、それでは友達に見せることはできません。
インターネットという大空に、自分の作品を放つ作業がデプロイです。
一度デプロイしてしまえば、スマホからでも、地球の裏側からでもあなたのサイトが見られるようになるんですよ！
-->

## 5-1. 豆知識：無料でおしゃれに公開！

今の時代、Webサイトを公開するのに1円もかかりません。

- **Vercel (バーセル)** / **Netlify (ネットリファイ)** / **GitHub Pages**:
  - GitHubにコードを置くだけで、自動でサイトを公開してくれる超便利サービス。
  - プロも愛用するほどの高性能なのに、個人の練習ならずっと無料！

**💡 これだけで:** 
- `https://your-site.vercel.app` のようなカッコいいURLが手に入ります。

<!-- 
【講師用台本】
昔はサーバーを借りて月額料金を払って…と大変でしたが、今は素晴らしいサービスがたくさんあります。
特におすすめなのが Vercel です。
GitHubと連携しておけば、皆さんが `git push` するたびに、自動で最新のサイトに更新してくれるんです。
まさに「全自動公開マシン」ですね！
-->

## 6-1. 【やってみよう 1】最初のセーブをしてみよう！

1. ターミナル（黒い画面）を開く。
2. 順番にコマンドを打ってみる。

```bash
git add .
git commit -m "ダッシュボードを公開するぞ！"
git push origin main
```

**✅ 成功イメージ:** GitHubのページをリロードして、自分のコードが表示されたら大成功！

## 8. 迷った時のチェックリスト

1. **GitHubの登録:** メール認証は済んでいますか？
2. **スペルミス:** `comit`（mが足りない）などはよくあるミス。
3. **初期設定:** 最初の1回だけ、名前やメールアドレスを登録する必要があります。
4. **反映待ち:** デプロイには数秒〜数分かかることがあります。焦らず待とう！

<!-- 
【講師用台本】

「こうすればいいよ！」というヒントをGitが教えてくれていることが多いですよ。
