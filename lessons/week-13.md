## 1-1. レストランに例える：Server vs Client

Next.js (App Router) では、部品ごとに「どこで動くか」を決められます。

- **Server Components（厨房）:**
  - データの取得、複雑な計算、APIキーの管理。
  - **お客様（ユーザー）の目に見えないところで準備をする場所。**
- **Client Components（客席＝'use client'）:**
  - ボタンクリック、フォーム入力、Stateの変更。
  - **お客様が直接触れるインタラクティブな場所。**

<!-- 
【講師用台本】
Webサイトをレストランだと思ってください。
これまでのJavaScriptは、客席にお客さんが座ってから、その場で料理（データ取得）を始めるようなものでした。
Next.jsのサーバーコンポーネントは違います。
厨房（サーバー）でお客さんが来る前に料理をほぼ完成させておいて、座った瞬間にドン！と提供する。
だから表示が爆速なんです！
-->

## 2-1. なぜサーバー側でデータを取るの？

いいことがたくさんあります！

- **表示が速い:** ブラウザで重いJSを動かす必要がないから。
- **安全:** 大事なAPIキーを、ユーザー（ブラウザ）から隠せる。
- **SEOに強い:** Googleのロボットが来た時に、すでにデータが入った状態を見せられる。

<!-- 
【講師用台本】
一番のメリットは「セキュリティ」と「スピード」です。
Phase 2の時は、ブラウザのソースコードを見ればAPIキーがバレてしまう危険がありましたが、
サーバー側で通信を行えば、その心配はゼロになります。
また、スマホの電波が弱くても、強力なサーバーが代わりに通信してくれるので、表示がとてもスムーズになるんですよ。
-->

## 3-1. 驚きの書き方：Async Components

Next.jsなら、コンポーネントそのものを `async` にできます！

```tsx
// app/page.tsx (デフォルトは Server Component)
export default async function Home() {
  // サーバー側でいきなり fetch！ (await が使える)
  const res = await fetch("https://api.github.com/users/your-name");
  const data = await res.json();

  return (
    <main>
      <h1>{data.name}のダッシュボード</h1>
    </main>
  );
}
```

<!-- 
【講師用台本】
これ、Reactの常識を覆す凄い機能なんです。
以前は `useEffect` という難しい仕組みを使わないとデータが取れませんでしたが、
Next.jsでは、関数そのものに `async` をつけるだけで、上から順番にデータを取ってこれるようになりました。
まるで普通のJavaScriptを書いているような感覚で、高度な通信が実現できるんですね。
-->

## 4-1. 自動で「おもてなし」：`loading.tsx`

データ取得中に表示される「読み込み中画面」を、ファイル一個で作れます。

- **作り方:** 
  - `app/` フォルダの中に `loading.tsx` という名前でファイルを作るだけ。
- **仕組み:**
  - `page.tsx` がデータを取っている間、Next.jsが自動でこのファイルを表示してくれます。

<!-- 
【講師用台本】
「ローディング画面を作るのって、Stateを管理して…面倒くさそう」
そう思ったあなたに朗報です。
`loading.tsx` というファイルを用意して、そこに「読み込み中です」と書いておくだけ。
これだけで、データの準備ができるまでの間、Next.jsが気を利かせてその画面を出しておいてくれます。
エンジニアの「おもてなしの心」が、こんなに簡単に形にできるんです！
-->

## 5-1. 豆知識：スケルトンスクリーン

今のトレンドは、「読み込み中...」という文字だけではありません。

- **Skeleton Screen:**
  - これから表示される情報の「影（ワク）」を先に見せる手法。
  - **メリット:** ユーザーは「あ、ここにニュースが出るんだな」と予測できるので、待ち時間が短く感じられます。

**💡 ヒント:** 
- `loading.tsx` の中で、グレーの四角形を並べて作ってみましょう！

## 8. 迷った時のチェックリスト

3. **ファイル名:** `loading.tsx` の綴りは正しいですか？
4. **Client vs Server:** `useState` を使っている部品を、間違えて Server Component のファイルに書いていませんか？
